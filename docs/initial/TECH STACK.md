# 대화 중심 외국어 학습 앱 - 기술 스택 결정

## 프로젝트 개요

- **앱 유형**: 대화 중심 언어 학습 앱 (듀오링고 스타일, 대화에 치중)
- **팀 규모**: 1인 / 소규모
- **개발 방식**: GenAI 협업

---

## 기술 스택 선택지 검토

### 네이티브 (Swift/Kotlin) - 탈락

**장점:**

- 플랫폼 API 직접 접근
- 최상의 성능

**탈락 이유:**

- 코드베이스 2개 유지 → 기능 추가마다 양쪽 수정 필요
- 버그 2배, 테스트 2배
- GenAI가 코드 작성은 도와줘도 유지보수 복잡도는 해결 못함
- 대화 앱은 AI API 호출이 병목이지, 앱 자체 성능은 크게 중요하지 않음

### Flutter - 후보

**장점:**

- 애니메이션 성능 우수
- 단일 렌더링 엔진 → 플랫폼 간 일관된 UX

**탈락 이유:**

- Dart 언어 학습 비용
- 기존 JS/TS 경험 활용 불가

### React Native + Expo - 선택 ✓

**선택 이유:**

- 코드 한 벌로 iOS/Android 동시 지원
- JS/TS 생태계 → AI API 연동 라이브러리 풍부
- Expo EAS Build → 네이티브 빌드 환경 없이 배포 가능
- 도그푸딩 시 iOS/Android 양쪽 기기에서 테스트 가능

---

## 최종 기술 스택

| 영역           | 선택                        | 비고                      |
| -------------- | --------------------------- | ------------------------- |
| 프레임워크     | Expo (managed workflow)     | React Native 기반         |
| 상태관리       | Zustand 또는 React Query    |                           |
| 백엔드         | Supabase                    | 인증 + DB + 실시간 동기화 |
| AI 대화        | Claude API                  |                           |
| 음성인식 (STT) | Whisper API                 |                           |
| 음성합성 (TTS) | expo-speech 또는 ElevenLabs |                           |

---

## AI 파이프라인 구조

```
사용자 음성 → STT (Whisper) → LLM (Claude) → TTS → 재생
                              ↓
                    문법 피드백, 대화 생성
```

---

## 개발 순서 (예정)

1. 프로젝트 세팅 + 기본 구조
2. AI 대화 프로토타입 (텍스트)
3. 음성 입출력 연동
