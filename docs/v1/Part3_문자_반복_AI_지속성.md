## 문자 체계

### 원칙

문자를 가르치지 않는다. 회화 상황 안에서 반복 노출되면서 **부산물로** 읽기 능력이 생긴다. 별도 あいうえお 학습 모듈 없음.

### 히라가나: 앰비언트 노출

처음부터 화면에 있되, 읽으라는 압력 제로.

```
  🔊 ふたり
     [?]           ← 탭하면 "2명" 3초 표시 후 소멸
```

사용자는 🔊로 소리를 듣고, [?]로 뜻을 확인한다. "ふたり"를 읽을 줄 알 필요 없다. 하지만 같은 소리에 같은 문자가 반복 노출되면서 연결이 쌓인다.

**전체 단어 인식이 먼저, 개별 문자 분석은 나중:**

```
ふたり (2명)  ─┐
ふね   (배)   ─┤── 사용자가 "ふ" 패턴을 자연 추출
ふゆ   (겨울) ─┘
```

**빈도 낮은 글자 해결**: 시나리오 추천 시 내부적으로 글자 커버리지 최적화. 사용자에게는 "새로운 상황 추천"일 뿐.

### 가타카나: 외래어 활용

```
  🔊 ビール
     [?]           ← "맥주 (beer)" 3초 표시
```

원어 힌트 "(beer)"가 외래어 패턴 인식을 돕는다.

### 한자: 환경 요소로만

회화 자막에 한자 사용 안 함. 메뉴판, 역 표지판 등 **상황 내 배경**으로 등장.

```
메뉴판:                     역명:
  ラーメン ... 800円          🚃 渋谷
  🔊 [?]                        🔊 [?]
  餃子 ....... 400円
  🔊 [?]
```

한자를 못 읽어도 🔊와 [?]로 소리와 뜻 확인 가능. 여행 상황의 사실적 재현이 목적이지, 한자 교육이 아님.

### [?] 안전망의 점진적 후퇴

| 노출 횟수 | [?] 탭 시 |
|-----------|-----------|
| 1~5회 | 탭 즉시 → 한국어 뜻 3초 표시 |
| 6~10회 | 🔊 "먼저 소리를 다시 들어보세요" → 3초 후 한국어 |
| 11회~ | 그림 힌트만 표시. 한국어는 2차 탭 |

단어별 독립 추적. 이 변화를 사용자에게 알리지 않는다.

### 문자 인식 체크 (간헐적)

포착에 10~20% 비율로 삽입: 음성 없이 히라가나만 보여주고 그림 고르기.

```
  (음성 없음)
       ふたり
  🧑    🧑🧑    🧑🧑🧑
  
  💡 소리가 필요하면 탭하세요
```

못 맞혀도 패널티 없음. 내부적으로 "아직 안전망 유지"로 판단.

## 반복 학습

### 원칙

"복습"이라는 단어를 쓰지 않는다.

### 같은 장소, 다른 시나리오

| 장소 | 기본 | 변주 1 | 변주 2 | 변주 3 |
|------|------|--------|--------|--------|
| 식당 | 기본 주문 | 없는 메뉴 질문 | 알레르기 확인 | 친구 것 대신 주문 |
| 편의점 | 물건 구매 | 택배 보내기 | ATM 위치 | 전자레인지 부탁 |
| 전철 | 목적지 이동 | 환승 묻기 | IC카드 충전 | 분실물 문의 |
| 호텔 | 체크인 | 체크아웃 변경 | 룸서비스 | 주변 식당 추천 |

이전 시나리오의 **오답 선택지가 다음 시나리오의 정답**이 되는 구조. 한 번의 활동에서 3개 표현이 동시에 노출 → 변주에서 각각 정답으로 재활용.

**UX**: 기본 완료 후 해당 장소에 "새로운 상황 발생!" 배지.

### 교차 상황 반복: 다의어 깨기

같은 표현이 여러 상황에서 다른 기능으로 등장. L1 번역어 하나에 고정되지 않는다.

**すみません 예시**:
- 공항: 직원에게 말 걸기 → [손 드는 그림]
- 전철: 양해 구하기 → [좁은 통로 그림]
- 식당: 점원 호출 → [점원 부르는 그림]
- 거리: 부딪힌 후 사과 → [고개 숙이는 그림]

4번의 노출이 **"실례합니다"라는 고정 번역**이 아닌, **여러 상황에서 쓰는 쿠션 표현**이라는 L2 고유 개념 범주를 형성.

### 간격 반복 타이밍 (내부)

1일 → 3일 → 7일 → 14일 → 30일. "오늘의 추천 상황"으로 유도. 로직 비노출.

## AI 대화 파트너

### 행동 규칙

| 규칙 | 내용 |
|------|------|
| 역할 유지 | 항상 NPC 역할. "그건 ～문법이에요" 안 함 |
| 난이도 적응 | 수준에 맞게 어휘/복잡도 자동 조절 |
| 피드백 계층화 | 리캐스트 / 명확화 요청 / 메타언어적 힌트 배합 |
| 예상 밖 대응 | 식당에서 "トイレどこ?" → 화장실 안내 |
| i+1 삽입 | 모를 법한 표현 하나씩 자연스럽게 |
| 대화 종결 | 5~7턴에 자연스럽게 마무리 |

### 선택지 생성

- 정답 1 + 올바른 일본어 오답 2
- 문법 오류를 선택지에 넣지 않음 — 잘못된 형태가 기억에 각인되는 것 방지

## 공통 도구 세트

상황을 관통하는 공통 표현을 별도 "도구 세트"로 추적.

| 카테고리 | 표현 예시 | 도입 상황 |
|----------|----------|----------|
| 인사 | こんにちは, ありがとう, すみません | 공항 |
| 숫자 | ひとつ~とお, ひゃく, せん | 전철 |
| 질문 | ～はどこですか, ～はいくらですか | 공항/편의점 |
| 요청 | ～をください, ～をおねがいします | 편의점/호텔 |
| 확인 | はい, いいえ, だいじょうぶです | 전 상황 |
| 긴급 | たすけてください, いたい | 긴급상황 |

별도 학습 메뉴 없음. 상황 내 자연 포함 + 하이라이트 + 리뷰에서 🔧 태그. 메인 지도 하단에 "나의 도구 세트" 모아보기.

## 지속성 설계

### 알림

| 유형 | 예시 | 톤 |
|------|------|-----|
| 일일 | "편의점에서 새로운 상황이 기다리고 있어요" | 여행 내러티브 |
| 연속 | "3일째 도쿄 여행 중!" | 격려 |
| 복귀 | "도쿄가 그리워요. 잠깐 들러볼까요?" | 가볍게 |

⛔ "스트릭이 끊겼습니다!", "목표 미달성" 등 죄책감 유발 알림 금지.

### 세션 완결감

- 종료 시: "오늘 당신은 식당에서 주문을 할 수 있게 되었습니다 🍜"
- 중도 이탈 시 진행 저장 → "지난번에 주문 중이었어요. 이어서 할까요?"

### 여행 D-Day (선택적)

출발일 등록 시 남은 일수에 맞춰 학습 순서/강도 자동 조절.
- D-30: 전 상황 한 번씩
- D-7: 핵심 상황 집중 반복
- D-1: 핵심 표현 10개 요약 카드

## Anti-Patterns (하지 않는 것)

| 배제 | 이유 |
|------|------|
| 레벨 / XP / 리더보드 | "학습 게임" 멘탈 모델 유발 |
| 스트릭 끊김 경고 | 죄책감 기반 동기 역효과 |
| "틀렸습니다" 메시지 | 시험 멘탈 모델. 피드백 계층화로 대체 |
| 점수 / 백분율 | 능력 기반 서술로 대체 |
| 단어장 암기 | 맥락 없는 암기. 상황 내 습득으로 대체 |
| 독립 문법 강의 | 체험 후 정리용으로만 |
| 한국어 발음 표기 | L1 음운 매핑 → 발음 화석화 |
| 별도 문자 학습 모듈 | 상황 노출로 침전 |
| L1 번역 상시 표시 | 경유지화 → 직접 경로 차단 |
| 받아쓰기 | 목표는 회화. 쓰기는 범위 밖 |
