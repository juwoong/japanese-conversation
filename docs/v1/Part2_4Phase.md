# 일본어 여행 회화 앱 — MVP 기획서 v2

> v2.0 | 2026.02.17 | Part 2/3: 4 Phase 상세 (관찰 · 포착 · 참여 · 정리)

---

## 전체 구조

하나의 상황(예: 식당)은 4 Phase를 순환한다. 총 세션 10~15분.

```
관찰 (30초~1분)  →  포착 (1.5~2.5분)  →  참여 (5~8분)  →  정리 (2~3분)
 Watch               Catch               Engage            Review

 소리 풍경 노출       소리-의미 직접      대화 실전          패턴 정리
 상황 맥락 형성       연결 형성           산출 40%+          발음 피드백
```

Phase 사이에 로딩 화면이나 단계 안내 없음. 하나의 연속된 경험.

---

## Phase 1 — 관찰 (Watch) | 30초~1분

### 하는 일

모델 대화를 보고 듣는다. "이 장소에서 이런 소리가 오간다"를 체험.

### 화면

```
┌──────────────────────────────────────┐
│  [식당 내부 일러스트]                 │
│                                      │
│  👤 🔊 いらっしゃいませ!              │
│        なんめいさまですか?             │
│                                      │
│  🧑 🔊 ふたりです。                   │
│                                      │
│  👤 🔊 こちらへどうぞ。               │
│                                      │
│  [💬 한국어로 보기]                   │
│                                      │
│  ▶ 다시 듣기          [다음으로 →]   │
└──────────────────────────────────────┘
```

### 규칙

- 대화가 한 줄씩 자동 순차 재생
- 자막은 **히라가나만** (한자 ✕, 한국어 발음 ✕)
- 핵심 표현은 살짝 다른 색상으로 하이라이트
- 말풍선 탭 → 해당 줄만 다시 재생
- 발화 속도: 네이티브 자연 속도. 핵심 부분만 약간 또렷하게

### [💬 한국어로 보기] — L1 안전망

탭하면 각 줄 옆에 한국어 뜻이 **회색, 3초간** 표시 후 자동 소멸.

```
  👤 いらっしゃいませ!
     なんめいさまですか?
                         ← (회색) 어서오세요! 몇 분이세요?
```

**영구 표시가 아닌 이유**: 한국어가 항상 보이면 눈이 거기로 간다. 매번 한국어를 거쳐서 의미에 도달하는 경로가 굳어짐. 일시 표시는 막힐 때만 확인하는 안전망.

### 모델 대화 제약

- 4~6턴 이내 (30초~1분). 길면 수동 시청이 되어 집중력 저하
- 핵심 표현 3~5개. 7개 이상이면 작업 기억 초과

## Phase 2 — 포착 (Catch) | 1.5~2.5분

### 하는 일

모델 대화에서 핵심 소리를 건져올린다. **소리→그림 직접 연결**(한국어 경유 없이)을 만들고, 조음 연습을 시작한다.

### 진입 전환

모델 대화가 끝나면 말풍선이 위로 흐려지고, 핵심 표현이 아래로 떨어져 나온다. 배경 일러스트는 유지 — 맥락이 끊기지 않음.

### 시퀀스 (한 세션)

```
1. 청크 캐치 (수용)    ~20초   전체 발화 듣고 상황 그림 고르기
2. 단어 캐치 (수용)    ~30초   핵심 단어 듣고 그림 고르기
3. 소리 구별 (수용)    ~15초   최소 대립쌍 듣기 (간헐적)
4. 따라 말하기 (산출)  ~20초   소리 듣고 따라 발화
5. 그림→말하기 (산출)  ~25초   그림 보고 해당 단어/청크 발화
```

### ① 청크 캐치

```
┌──────────────────────────────────────┐
│  🔊 (점원의 전체 발화 재생)           │
│                                      │
│  이 말은 어떤 상황일까요?             │
│                                      │
│  ┌────────┐ ┌────────┐ ┌────────┐   │
│  │🍽 인원을│ │🛒 봉투를│ │💴 계산 │   │
│  │ 묻는    │ │ 묻는    │ │ 하는   │   │
│  └────────┘ └────────┘ └────────┘   │
└──────────────────────────────────────┘
```

문장 전체를 하나의 소리 덩어리로 듣고 상황 그림을 고른다. 단어별 해독이 아닌 **청크 단위 인식**. 실전 회화에서의 처리 단위가 청크이기 때문.

### ② 단어 캐치

```
┌──────────────────────────────────────┐
│  🔊 (ふたり만 재생)                   │
│                                      │
│  ┌────────┐ ┌────────┐ ┌────────┐   │
│  │  🧑    │ │ 🧑🧑  │ │🧑🧑🧑 │   │
│  └────────┘ └────────┘ └────────┘   │
└──────────────────────────────────────┘
```

정답 시: 🔊 ふたり + [?] 표시. 한국어 없이 **소리→그림** 직접 연결이 기본.

| 단어 유형 | 예시 | 매칭 방식 |
|-----------|------|-----------|
| 수량 | ふたり, みっつ | 소리 → 숫자 시각화 |
| 구체 명사 | ビール, ラーメン | 소리 → 사물 그림 |
| 행위/인사 | いらっしゃいませ | 소리 → 상황 애니메이션 |

### ③ 소리 구별 (간헐적)

해당 상황에 음소 구별이 중요한 쌍이 있을 때만 삽입.

```
  🔊 A: おばさん     🔊 B: おばあさん

  같은 말일까요?   [같다] [다르다]

  → 맞아요!
    🔊 おばさん   → [아주머니 그림]
    🔊 おばあさん → [할머니 그림]
    소리가 살짝 길면 뜻이 달라져요
```

뜻 전달은 한국어 단어가 아닌 **그림**으로. 메타 힌트("소리가 길면~")가 L1/L2 차이 인식을 만든다.

### ④ 따라 말하기

🔊 재생 → 사용자 따라 발화. 단어 하나라서 발음 분석 정확도 높음.  
묵음 모드: "머릿속으로 해보기" 탭 → 정답 음성 재생.

### ⑤ 그림→말하기

점원 질문 음성 + 상황 그림 → 사용자가 발화. 포착의 마지막 활동이 참여의 예고편.

### 재방문 시 변화

| 방문 | 포착 구성 |
|------|----------|
| 1회 | 풀 세트 |
| 2회 | 단어 캐치 생략, 산출부터 |
| 3회+ | 변주 시나리오의 새 단어만 |
| 숙련 | "바로 대화로" 스킵 가능 |

## Phase 3 — 참여 (Engage) | 5~8분

### 하는 일

AI NPC와 실전 대화. **첫 방문에서도 산출 비율 40% 이상.**

### 입력 난이도 단계

```
[첫 시도]            [2~3회차]            [숙련]
선택지 + 따라말하기  →  빈칸 채우기       →  자유 입력
  ★                      ★★                 ★★★
```

### 첫 방문 턴 구성 (식당 예시)

| 턴 | NPC | 사용자 | 유형 |
|----|-----|--------|------|
| 1 | なんめいさまですか? | 3개 선택지 중 택1 | 수용 |
| 2 | — | "직접 말해보세요" → 따라 말하기 | **산출** |
| 3 | メニューです。 | 메뉴판에서 항목 탭 | 수용 |
| 4 | ごちゅうもんは? | "라면으로 할게요, 라고 말해보세요" | **산출** |
| 5 | おのみものは? (i+1) | 선택지 + 따라 말하기 | 수용→**산출** |
| 6 | しょうしょうおまちください | (대화 종결) | 수용 |

→ 6턴 중 3턴 산출 = 50%

### 선택지 화면

```
┌──────────────────────────────────────┐
│  👤 🔊 なんめいさまですか?            │
│                                      │
│  당신은 2명입니다. 뭐라고 할까요?     │
│                                      │
│  ┌──────────────────────────────┐    │
│  │  🔊 ふたりです               │    │
│  ├──────────────────────────────┤    │
│  │  🔊 ひとりです               │    │
│  ├──────────────────────────────┤    │
│  │  🔊 さんにんです             │    │
│  └──────────────────────────────┘    │
└──────────────────────────────────────┘
```

**선택지 규칙**:
- 히라가나 + 음성으로만 구성 (한국어 없음)
- 오답도 문법적으로 맞는 일본어 (문법 오류를 선택지에 넣지 않음)
- 오답이 오답인 이유는 "상황이 다를 뿐" — 변주 시나리오에서 정답이 됨
- 🔊 탭으로 선택 전에 발음 미리 듣기 가능

### 선택 후 따라 말하기

```
  ✅ 대화가 이어집니다...
  
  이제 직접 말해보세요
  🔊 ふたりです
  [🎤 녹음 시작]
```

### 빈칸 채우기 (2~3회차)

```
  👤 🔊 なんめいさまですか?
  
  [____] にんです
  
  키보드: ひとり  ふたり  さんにん
```

### 자유 입력 (숙련)

```
  👤 🔊 なんめいさまですか?
  
  [🎤 음성으로 답하기]
  [⌨ 텍스트로 입력하기]
```

### 산출의 단계적 구조

| 단계 | 활동 | 인지적 요구 | 위치 |
|------|------|------------|------|
| 1 | 따라 말하기 | 조음만 | 포착 |
| 2 | 프롬프트 산출 ("～라고 말해보세요") | 형식화 + 조음 | 포착~참여 |
| 3 | 상황 산출 (그림+질문 → 발화) | 개념화(부분) + 형식화 + 조음 | 참여 |
| 4 | 자유 산출 | 전부 | 참여 후반 |

### 난이도 전환 기준

| | 선택지 → 빈칸 | 빈칸 → 자유 |
|--|---------------|-------------|
| 정답률 | 80%+ (2회 연속) | 80%+ (2회 연속) |
| 방문 횟수 | 3회+ | 5회+ |

한 세션 안에서 섞어서 전환. "갑자기 어려워졌다"를 방지.

## 피드백 계층화

"틀렸습니다" 없음. 오류 유형별 자연스러운 반응.

### 3단계 분기

```
오류 발생
  ├─ 의미 통하는 형태 오류 → ① 리캐스트
  ├─ 의미 불통 → ② 명확화 요청
  └─ 같은 실수 2회+ → ③ 메타언어적 힌트
```

### ① 리캐스트 (강화)

| 사용자 | NPC |
|--------|-----|
| ビール ふたつ ください | ビールを ✨ふたつ✨ ですね! しょうしょうおまちください |

교정 부분에 살짝 다른 배경색 + ✨. 탭하면 "이렇게도 말해요" 팝업.

### ② 명확화 요청

```
사용자: (불명확한 발화)
점원: すみません、もういちどおねがいします
→ 사용자 재시도 유도
```

### ③ 메타언어적 힌트

같은 오류 2회 이상일 때만. 대화 잠깐 멈추고:

```
💡 숫자 뒤에 つ를 붙이면 물건을 셀 수 있어요
   🔊 ひとつ、ふたつ、みっつ...
   [계속하기]
```

## Phase 4 — 정리 (Review) | 2~3분

### 화면

```
┌──────────────────────────────────────┐
│  ✅ 식당에서 주문 완료!               │
│                                      │
│  🔊 なんめいさまですか?              │
│     [?]  [왜 이렇게 말할까? ▾]       │
│                                      │
│  🔊 ～にします                       │
│     [?]  [왜 이렇게 말할까? ▾]       │
│                                      │
│  🔊 おかいけいおねがいします          │
│     [?]  [왜 이렇게 말할까? ▾]       │
│                                      │
│  (발음 피드백 — 음성 입력 시에만)     │
│                                      │
│           [지도로 돌아가기]           │
└──────────────────────────────────────┘
```

### [왜 이렇게 말할까?]

"문법 보기"가 아니라 호기심 프레이밍.

```
식당에서 메뉴를 골랐을 때, に는 '이걸로!'라는
선택의 느낌이에요. します를 붙이면 정중해져요.

🔊 ラーメンにします   (라면으로 할게요)
🔊 ビールにします     (맥주로 할게요)
🔊 これにします       (이걸로 할게요)

💡 に 뒤에 뭘 넣어도 같은 패턴이에요.
```

**규칙**:
- 문법 용어 없음
- 예문 3개 이상 (패턴 추출 최소 수)
- 전부 같은 상황 안의 예문 (맥락 이탈 없음)
- 마지막 줄에서 생산성 암시 ("뭘 넣어도~")
- 뜻은 괄호 안 간략 제공

### 점수/평가 없음

- ⛔ "85점", "정답률 70%"
- ✅ "이 상황을 혼자 해결할 수 있어요"
